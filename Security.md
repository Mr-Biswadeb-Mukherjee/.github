# Security Policy

## Overview

Biswadeb Mukherjee's Lab publishes research projects related to offensive security, malware engineering, reconnaissance, and adversarial simulation. These projects are intended for **educational purposes, authorized security testing, and cybersecurity research**.

The tools and frameworks released by this organisation are designed to help researchers and security professionals better understand adversarial techniques, system weaknesses, and defensive strategies.

---

## Responsible Use

All software and research materials provided by this organisation must be used **only in lawful and authorised environments**.

Acceptable use includes:

* security research
* academic study
* penetration testing with explicit authorisation
* defensive security analysis
* laboratory experimentation

The use of these tools against systems without permission, or for malicious or illegal activities, is strictly discouraged.

The maintainers of this organisation assume **no responsibility for misuse** of the software published here.

---

## Reporting Security Issues

If you discover a vulnerability or security issue related to any project within this organisation, please report it responsibly.

Please include the following information when reporting a vulnerability:

* affected project or repository
* description of the vulnerability
* Steps to reproduce the issue
* potential impact
* proof-of-concept (if available)

Reports can be submitted via:

**Email:** [admin@official-biswadeb941.in](mailto:admin@official-biswadeb941.in)

We request that security issues are **not publicly disclosed until they have been reviewed and addressed**.

---

## Disclosure Policy

Once a reported issue has been validated, the maintainer will work toward:

1. Confirming the vulnerability
2. developing a fix or mitigation
3. releasing an update or advisory

Responsible disclosure helps maintain the integrity of the research and protects users of the software.

---

## Scope

This policy applies to all repositories maintained under this organisation unless a specific repository defines its own security policy.
